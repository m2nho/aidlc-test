# User Personas

테이블오더 서비스의 주요 사용자 유형을 정의합니다.

---

## Persona 1: 고객 (Customer)

### 기본 정보
- **이름**: 김민준
- **나이**: 28세
- **직업**: 회사원
- **역할**: 매장 방문 고객
- **기술 수준**: 중급 (스마트폰 및 태블릿 사용 익숙)

### 목표 (Goals)
1. **빠른 주문**: 메뉴를 보고 빠르게 주문하고 싶음
2. **편리한 경험**: 직원을 부르지 않고도 자유롭게 추가 주문
3. **투명한 내역**: 자신이 무엇을 주문했는지 언제든지 확인
4. **정확한 주문**: 원하는 메뉴를 정확하게 주문

### 동기 (Motivations)
- 직원을 부르기 위해 기다리는 시간 감소
- 메뉴를 천천히 보면서 선택하고 싶음
- 주문한 내역을 시각적으로 확인하여 실수 방지
- 원하는 시점에 자유롭게 추가 주문

### 불편사항 (Pain Points)
- 직원이 바쁠 때 주문하기 어려움
- 메뉴판이 멀리 있거나 보기 어려움
- 주문 후 무엇을 시켰는지 기억이 안 남
- 추가 주문할 때마다 직원을 불러야 함
- 주문 실수로 인한 불편함

### 사용 환경 (Context)
- **장소**: 식당 테이블
- **디바이스**: 테이블에 비치된 태블릿
- **사용 시간**: 식사 시간 (점심/저녁)
- **동반자**: 혼자 또는 친구/동료와 함께
- **상황**: 빠르게 주문하고 싶거나, 천천히 메뉴를 고르고 싶은 경우

### 주요 니즈 (Needs)
- 직관적이고 사용하기 쉬운 UI
- 메뉴 이미지와 설명을 명확히 볼 수 있어야 함
- 장바구니에서 주문 전 확인 가능
- 주문 내역을 언제든지 조회 가능
- 빠른 주문 처리 (2초 이내 응답)

## Persona 2: 관리자 (Admin)

### 기본 정보
- **이름**: 이서연
- **나이**: 35세
- **직업**: 매장 운영자 / 점장
- **역할**: 주문 관리 및 매장 운영
- **기술 수준**: 중급 (일반적인 관리 시스템 사용 경험)

### 목표 (Goals)
1. **효율적인 주문 관리**: 들어오는 주문을 실시간으로 확인하고 처리
2. **테이블 상태 파악**: 각 테이블의 주문 상황과 총액을 한눈에 파악
3. **빠른 대응**: 새 주문이 들어오면 즉시 확인하고 주방에 전달
4. **정확한 정산**: 테이블별 주문 내역과 총액 정확히 관리
5. **매장 운영 최적화**: 인건비 절감 및 서비스 품질 향상

### 동기 (Motivations)
- 주문 누락 방지로 고객 만족도 향상
- 직원의 주문 받기 업무 감소로 다른 서비스에 집중
- 정확한 주문 관리로 실수 최소화
- 실시간 모니터링으로 빠른 의사결정
- 매출 및 주문 데이터 정확한 기록

### 불편사항 (Pain Points)
- 주문이 많을 때 놓치는 경우 발생
- 테이블별 주문 상황을 파악하기 어려움
- 직원이 주문을 구두로 전달할 때 실수 발생
- 주문 수정이나 취소 처리가 복잡함
- 테이블 회전율 관리 어려움
- 과거 주문 내역 조회가 불편함

### 사용 환경 (Context)
- **장소**: 매장 내부 (카운터 또는 주방 근처)
- **디바이스**: 태블릿 또는 PC
- **사용 시간**: 매장 운영 시간 (장시간 지속적 사용)
- **상황**: 다수의 테이블을 동시에 관리
- **멀티태스킹**: 주문 관리 외에 다른 매장 업무 병행

### 주요 니즈 (Needs)
- 실시간 주문 업데이트 (SSE, 2초 이내)
- 테이블별 주문 현황 한눈에 파악 (대시보드)
- 신규 주문 시각적 강조 (색상, 애니메이션)
- 주문 상태 변경 간편 (대기중/준비중/완료)
- 주문 삭제 및 수정 기능 (오류 정정)
- 테이블 세션 종료 기능 (정산 후 초기화)
- 과거 주문 내역 조회 (날짜별 필터)
- 메뉴 관리 기능 (추가/수정/삭제)
- 16시간 세션 유지 (자주 로그인하지 않아도 됨)

## Persona 간 관계 및 상호작용

### 고객 → 시스템 → 관리자
```
[고객] 주문 생성
    ↓
[시스템] 주문 저장 및 실시간 전송 (SSE)
    ↓
[관리자] 주문 수신 및 확인
    ↓
[관리자] 주문 상태 변경 (준비중 → 완료)
    ↓
[시스템] 상태 업데이트
    ↓
[고객] 주문 내역에서 상태 확인 (선택사항)
```

### 상호작용 포인트
1. **주문 생성 시점**: 고객이 주문하면 관리자에게 실시간 전달
2. **주문 상태 변경**: 관리자가 상태를 변경하면 시스템에 반영
3. **테이블 세션**: 고객의 세션은 관리자가 "이용 완료" 처리로 종료
4. **메뉴 정보**: 관리자가 관리하는 메뉴를 고객이 조회

### 비동기적 상호작용
- 고객과 관리자는 직접 소통하지 않음
- 시스템이 중재자 역할
- 실시간 데이터 동기화가 핵심

## Persona Summary

| 구분 | 고객 (Customer) | 관리자 (Admin) |
|---|---|---|
| **주요 목표** | 빠르고 편리한 주문 | 효율적인 주문 관리 |
| **핵심 니즈** | 직관적 UI, 빠른 응답 | 실시간 모니터링, 대시보드 |
| **사용 빈도** | 식사 시간 (30분~1시간) | 매장 운영 시간 (8시간+) |
| **기술 숙련도** | 중급 | 중급 |
| **주요 불편사항** | 직원 호출 대기 시간 | 주문 누락 및 실수 |
| **성공 기준** | 3번 터치 이내 주문 완료 | 모든 주문 2초 이내 확인 |

## Persona 활용 방법

각 User Story 작성 시 다음을 고려:
1. **어떤 Persona를 위한 Story인가?**
2. **해당 Persona의 목표를 달성하는가?**
3. **해당 Persona의 불편사항을 해결하는가?**
4. **해당 Persona의 사용 환경에 적합한가?**

이를 통해 실제 사용자 중심의 기능을 설계하고 구현합니다.
